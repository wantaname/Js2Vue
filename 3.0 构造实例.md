## 初始化实例

在Vue构造函数中，会调用

```js
this._init(options)
```

来初始化组件实例

初始化实例即在实例vm上添加属性(数据和方法)的过程

```js
vm._uid = uid++
```

组件id，每一个组件都有一个唯一的id

```js
vm._isVue = true
```

避免组件被observe【Vue中响应式对象首先要进行observe】

然后进行选项options的合并，即初始化vm.$options的过程。

```js
if (options && options._isComponent) {
    initInternalComponent(vm, options)
}else {
    vm.$options = mergeOptions(
        resolveConstructorOptions(vm.constructor),
        options || {},
        vm
      )
}
```

这里会判断是否是内部组件。因为子组件构造器extend Vue，会复用Vue实例的代码；对Vue实例与内部组件(子组件)的处理方式不同。

比如：

```js
<template id="t">
  <div :title="title">
    <div>{{title}}</div>
    <blog-post post-title="hello wolrd"></blog-post>
  </div>
</template>
```

```js
Vue.component('blog-post', {
    props: ['postTitle'],
    name:"hello",
    data:function (){
        return {
            "hello":"world"
        }
    },
    template:"<h1>{{postTitle}}</h1>"
})
var vm = new Vue({
    el: "#app"
    ...
}
```

这里的"blog-post"就是一个内部组件，而vm是Vue实例；在由vm的vnode生成真实node的过程中，遇到了子组件节点，则会去构造子组件实例。

按照代码执行顺序，我们这里只分析Vue实例的情况。

### 选项合并

合并构造器选项和参数选项，得到vm.$options属性

```js
vm.$options = mergeOptions(
    // 可以简单的认为它返回Vue.options，作为父options
    resolveConstructorOptions(vm.constructor),
    // 传入的options
    options || {},
    vm
)
```

在mergeOptions中，

```js
// child也可以是构造函数，这种情况下，child为child函数上的options属性
if (typeof child === 'function') {
    child = child.options
}
```

然后规范化三个属性：

```js
// 规范化options.props
normalizeProps(child, vm)
//规范化options.inject
normalizeInject(child, vm)
// 规范化directives
normalizeDirectives(child)
```

`normalizeProps`

会把数组形式的props转化为对象形式：

```js
props = {
    name:[PropOptions]
}
PropOptions = {
  type: Function | Array<Function> | null;
  default: any;
  required: ?boolean;
  validator: ?Function;
}
```

`normalizeInject`

```
Array<string> ==> { [key: string]: string | Symbol | Object }
```

`normalizeDirectives`

```js
/*
* 将{focus:function def(){},}转化为{focus:{bind:def,update:def}}
* */
function normalizeDirectives (options: Object) {
  //
  const dirs = options.directives
  if (dirs) {
    // directives是一个对象，每一个属性是一个函数
    for (const key in dirs) {
      const def = dirs[key]
      if (typeof def === 'function') {

        dirs[key] = { bind: def, update: def }
      }
    }
  }
}
```

规范化完成后，会处理child options中的extends和mixins混入：

```js
// 不是组件构造器中的options，即是选项对象，有extends和mixins属性
  if (!child._base) {
    // 如果有child.extends属性
    if (child.extends) {
      // 将parent和child.extends合并到parent上
      parent = mergeOptions(parent, child.extends, vm)
    }
    // mixins 选项接收一个混入对象的数组，即mixins是一个数组，每个数组元素是一个对象
    if (child.mixins) {
      for (let i = 0, l = child.mixins.length; i < l; i++) {
        // 把它们合并到parent
        parent = mergeOptions(parent, child.mixins[i], vm)
      }
    }
  }
```

然后，对parent和child中的每一个可枚举属性执行合并：

```js
// 最终合并后的选项
const options = {}
let key
// 对parent options中的每一个key，执行合并
for (key in parent) {
    mergeField(key)
}
// 对child options中的每一个key，执行合并
for (key in child) {
    if (!hasOwn(parent, key)) {
        mergeField(key)
    }
}
```

```js
function mergeField (key) {
    // 没有自定义合并策略就选默认策略
    const strat = strats[key] || defaultStrat
    options[key] = strat(parent[key], child[key], vm, key)
}
```

针对不同的属性，可能会有不同的合并策略：

`1. 默认合并策略`：

```js
// 默认合并策略：child有就选child，没有就选parent
const defaultStrat = function (parentVal: any, childVal: any): any {
  return childVal === undefined
    ? parentVal
    : childVal
}
```

`2. data合并策略`：

最终得到一个函数

```js
mergedInstanceDataFn () {
    // instance merge
    const instanceData = typeof childVal === 'function'
    ? childVal.call(vm, vm)
    : childVal
    const defaultData = typeof parentVal === 'function'
    ? parentVal.call(vm, vm)
    : parentVal
    if (instanceData) {
        return mergeData(instanceData, defaultData)
    } else {
        return defaultData
    }
}
```

执行这个函数才能得到data对象

`3. 生命周期函数合并策略`：

```js
// 12个生命周期函数
LIFECYCLE_HOOKS.forEach(hook => {
  strats[hook] = mergeHook
})

function mergeHook (
  parentVal: ?Array<Function>,
  childVal: ?Function | ?Array<Function>
): ?Array<Function> {
  const res = childVal
    ? parentVal
      ? parentVal.concat(childVal)
      : Array.isArray(childVal)
        ? childVal
        : [childVal]
    : parentVal
  return res
    ? dedupeHooks(res)
    : res
}
```

可以看到，生命周期钩子函数可以传入函数或者函数数组，最终对同一个钩子的函数会合并成数组

`4.资源合并策略`

```js
// options[type+'s']合并策略
ASSET_TYPES.forEach(function (type) {
  strats[type + 's'] = mergeAssets
})
function mergeAssets (
  parentVal: ?Object,
  childVal: ?Object,
  vm?: Component,
  key: string
): Object {
  // res的原型对象指针是parentVal
  const res = Object.create(parentVal || null)
  if (childVal) {
    process.env.NODE_ENV !== 'production' && assertObjectType(key, childVal, vm)
    //将childVal上的属性拿到res上
    return extend(res, childVal)
  } else {
    return res
  }
}
```

`5.watch`

watch的形式：

```js
{
	[key]:function(){}
}
```

对watch对象中的每一个key的函数，将它们合并成数组

```js
{
	[key]:[f1(),f2()...]
}
```

`6.其他`

```js
strats.props =
strats.methods =
strats.inject =
strats.computed = function (
  parentVal: ?Object,
  childVal: ?Object,
  vm?: Component,
  key: string
): ?Object {
  if (childVal && process.env.NODE_ENV !== 'production') {
    assertObjectType(key, childVal, vm)
  }
  if (!parentVal) return childVal
  const ret = Object.create(null)
  extend(ret, parentVal)
  if (childVal) extend(ret, childVal)
  return ret
}
```

比如：

```js
// parent
methods:{
	m1(){}
}
// child
methods:{
	m2(){}
}
//结果
// parent
methods:{
	m1(){},
	m2(){}
}
//子属性可能会覆盖父属性
```

### renderProxy

选项合并后，执行

```js
initProxy(vm)
```

```js
initProxy = function initProxy (vm) {
    if (hasProxy) {
      // determine which proxy handler to use
      // 暂存options
      const options = vm.$options
      const handlers = options.render && options.render._withStripped
        ? getHandler
        : hasHandler
      //vm._renderProxy代理了对vm的属性get或has
      vm._renderProxy = new Proxy(vm, handlers)
    } else {
      vm._renderProxy = vm
    }
  }
```

只捕捉了has操作，其他操作会无差别传递

#### initLifecycle

```js
// options的parent属性，是当前实例的父实例
let parent = options.parent
// 如果有父实例而且当前options不是抽象的
if (parent && !options.abstract) {
    // 找到第一个非抽象的父实例
    while (parent.$options.abstract && parent.$parent) {
        parent = parent.$parent
    }
    // 父实例的$children属性数组中加入vm
    parent.$children.push(vm)
}
// 当前实例vm的$parent属性
vm.$parent = parent
// $root属性，没有则指向自己
vm.$root = parent ? parent.$root : vm

// children
vm.$children = []
// refs
vm.$refs = {}
```

选项中的abstract表示是抽象组件，不会被渲染呈现在页面中，比如内置的keepAlive组件

选项的parent属性可以是构造实例时传入的；更多时候是使用内部组件时的选项，比如：

```js
// 1.内部组件选项
const options: InternalComponentOptions = {
    _isComponent: true,
    _parentVnode: vnode,//当前的vnode，为什么取名叫_parentVnode?
    parent
}
// 2.根据选项构建一个组件实例
return new vnode.componentOptions.Ctor(options)
```

vm.$parent是父组件实例的引用；同样也有vm.$children，保存了当前实例所有的子组件实例

`ref` 被用来给元素或子组件注册引用信息。引用信息将会注册在父组件的 `$refs` 对象上。如果在普通的 DOM 元素上使用，引用指向的就是 DOM 元素；如果用在子组件上，引用就指向组件实例：

#### beforeCreate钩子

当执行完上述初始化后，会调用生命周期钩子：

```js
callHook(vm, 'beforeCreate')
```

```js
export function callHook (vm: Component, hook: string) {
  // #7573 disable dep collection when invoking lifecycle hooks
  // 当激活生命周期钩子的时候禁止依赖收集
  pushTarget()
  // 从选项上拿到钩子函数数组
  const handlers = vm.$options[hook]
  const info = `${hook} hook`
  if (handlers) {
    //
    for (let i = 0, j = handlers.length; i < j; i++) {
      // 执行数组中的每一个钩子函数
      invokeWithErrorHandling(handlers[i], vm, null, vm, info)
    }
  }
  if (vm._hasHookEvent) {
    vm.$emit('hook:' + hook)
  }
  popTarget()
}
```
